Tarefas Futuras do Projeto OBD

Esta lista contém sugestões de melhorias e novas funcionalidades para o projeto, baseada no estado atual do desenvolvimento.

1. Interface e Experiência do Usuário (Frontend)

  • Polimento Mobile: Verificar e ajustar a responsividade de todas as páginas do dashboard em dispositivos móveis.
  • Correção de Lints: Resolver os avisos de sintaxe HTML/JS reportados pelo editor (ex: falta de ponto e vírgula, tags mal formatadas).
  • Feedback Visual: Implementar mensagens de sucesso/erro (toasts) para ações do usuário (ex: ao salvar perfil).

2. Funcionalidades Pendentes

  • Página de Ligas: Implementar a lógica e o layout da página "Ligas OBD" (/boa/leagues/), que atualmente exibe "Em construção".
  • Gestão de Documentos: Verificar se a seção de documentos precisa de upload/download dinâmico.
  • Média por Ano no Card: Implementar o cálculo e exibição da média (AVERAGE) do jogador filtrada por ano no Card do Membro.

3. Qualidade de Código e Backend

  • Testes Automatizados: Criar testes unitários para funcionalidades críticas (cálculo de estatísticas, geração de QR Code).
  • Refatoração: Otimizar views complexas em views.py para melhorar a manutenção.
  • Limpeza: Remover scripts de correção temporários (ex: fix_templates.py) se não forem mais necessários.

4. Performance e SEO

  • Otimização de Imagens: Verificar o tamanho das imagens carregadas (logos, banners) para garantir carregamento rápido.
  • SEO: Adicionar meta tags adequadas nas páginas públicas para melhor indexação nos buscadores.
